Semtex: HTML Templating Engine

Writing html is verbose, tedious and error prone.

Semtex offers an abridged and terse syntax that makes dealing with these kinds of structured documents a pleasure.
Whether in the browser, on the server or as a command-line tool, Semtex has been designed with simplicity in mind.

This project is still under pre-alpha development!

Which basically means it's new and I'm still working on it...

Usage

The first step is to define a template:

html:
  head:
    meta[name="keywords",content="HTML,CSS,XML,JavaScript"]:
    title: {{title}}
  body:
    p: Hello, {{message}}!
    div:
      lots of lovely content
      to be placed here...

Once the template source code has been defined, the next step is to compile it into a render function. The actual template can be loaded any way you desire, providing that when it enters the compile() function it’s a string:

var render = semtex.compile(src);

Now, all that’s left to do is render the template into html, passing in any variable replacement values:

var html = render({
  title: 'Semtex example',
  message: 'Semtex'
});

Which, given the template described above, will produce the following output:

<html>
  <head>
    <meta name="keywords" content="HTML,CSS,XML,JavaScript"/>
    <title>Semtex example</title>
  </head>
  <body>
    <p>Hello, Semtex!</p>
    <div>lots of lovely content to be placed here...</div>
  </body>
</html>

Basics

HTML, with its open-close tag structure and lax whitespace, ...TODO

Semtex takes a different approach. By using special syntactical conventions and whitespace-based hierarchies, it offers a more consistent and easy to read format.

At its lowest level, Semtex consists of tags and content.

Tags are identified by their trailing : character, for example html: defines a <html> tag. They can also be nested just like html, using a 2-space indent conventon:

html:
  head:
    title:
  body:
    h1:
<html>
  <head>
    <title></title>
  </head>
  <body>
    <h1></h1>
  </body>
</html>
